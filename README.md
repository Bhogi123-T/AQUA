## 📌 Overview

**AQUA** is a production-deployed, AI-driven Progressive Web App built to transform the global aquaculture industry. It provides a complete end-to-end digital ecosystem — connecting farmers, hatcheries, buyers, lab technicians, processing plants, and contractors — with state-of-the-art ML-powered predictions for yield, disease, stocking density, and market pricing.

<br/>

```
  WITHOUT AQUA                            WITH AQUA
  ────────────                            ──────────
  📋 Manual yield estimation              🤖 ADER predicts yield at 92–95%
  🦠 Disease spotted too late             🧬 APDC flags risk before outbreak
  🎲 Guessing stocking density            📊 ASER optimizes density at 90–93%
  📉 No market price visibility           💹 AMPRO forecasts price at 85–89%
  🤷 Siloed supply chain roles            🔗 7 stakeholders — one ecosystem
  📵 Desktop-only platforms               📱 PWA — offline, any device
```

> 🔴 **Production Live:** [aqua-ttiu.onrender.com](https://aqua-ttiu.onrender.com)

## 🚀 Quick Start

<br/>

### Prerequisites

| Requirement | Purpose |
|:---|:---|
| **Python 3.8+** | Backend runtime |
| **Supabase account** | Cloud database & auth *(optional — local JSON fallback available)* |
| **Twilio account** | SMS OTP notifications |
| **Gmail App Password** | Email OTP notifications |
| **Google OAuth credentials** | Social login |

<br/>

### Step 1 — Clone & Virtual Environment

```bash
git clone https://github.com/Bhogi123-T/AQUA.git
cd AQUA

# Create and activate virtual environment
python -m venv .venv
source .venv/bin/activate          # macOS / Linux
.venv\Scripts\activate             # Windows

pip install -r requirements.txt
```

<br/>

### Step 2 — Configure Environment Variables

```bash
cp .env.example .env
```

Edit `.env` with your credentials:

```env
# ─── Flask ───────────────────────────────────────────────────────
SECRET_KEY=your-strong-secret-key-here

# ─── Supabase ────────────────────────────────────────────────────
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-supabase-anon-key

# ─── Email OTP (Gmail) ───────────────────────────────────────────
MAIL_SERVER=smtp.gmail.com
MAIL_PORT=587
MAIL_USE_TLS=True
MAIL_USERNAME=your_email@gmail.com
MAIL_PASSWORD=your_app_specific_password

# ─── Google OAuth 2.0 ────────────────────────────────────────────
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret

# ─── Twilio SMS OTP ──────────────────────────────────────────────
TWILIO_ACCOUNT_SID=your_twilio_account_sid
TWILIO_AUTH_TOKEN=your_twilio_auth_token
TWILIO_PHONE_NUMBER=+1xxxxxxxxxx
```

<details>
<summary>💡 <b>How to get each key</b></summary>
<br/>

| Key | Where to Get |
|:---|:---|
| `SUPABASE_URL / KEY` | [supabase.com](https://supabase.com) → Project Settings → API |
| `MAIL_PASSWORD` | Google Account → Security → App Passwords |
| `GOOGLE_CLIENT_ID/SECRET` | [console.cloud.google.com](https://console.cloud.google.com) → OAuth 2.0 |
| `TWILIO_*` | [console.twilio.com](https://console.twilio.com) → Account Info |

</details>

<br/>

### Step 3 — Run Locally

```bash
python app.py
```

## 🌐 Deployment

<br/>

| Platform | Role | Status |
|:---|:---|:---:|
| **Render** | Full Flask backend + ML model serving | ✅ Live |
| **Supabase** | PostgreSQL database · Auth · Realtime | ✅ Active |
| **Vercel** | Static frontend *(if decoupled)* | ⚙️ Optional |

<br/>

### Deploy to Render

```
1. Push code to GitHub
2. Go to render.com → New Web Service
3. Connect your AQUA repository
4. Set Build Command  → pip install -r requirements.txt
5. Set Start Command  → python app.py
6. Add all .env variables in the Environment section
7. Click Deploy!
```
